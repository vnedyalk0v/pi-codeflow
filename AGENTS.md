# Agent Instructions

These rules apply to AI agents working in this repository.

## Branches

- Do not work directly on `main` or `dev` except for the initial bootstrap.
- Use semantic branches: `feat/`, `fix/`, `docs/`, `chore/`, `refactor/`, `test/`, or `ci/`.
- Prefer small PRs with one clear purpose.

## Workflow

- Do not invent commit or PR formats.
- Use the documented templates in `templates/` and `.github/pull_request_template.md`.
- Keep docs and issues consistent when behavior changes.
- Do not add production code without an issue.
- Do not add dependencies without explaining why.

## Reporting

Final reports must include:

- changed files
- checks run and results
- related issues
- known risks or follow-up work
