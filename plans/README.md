## Findings considered and rejected

- **Regex redaction misses bare high-entropy tokens** (tokens without a known
  prefix or `key: value` shape pass through `redactSecrets`): by design.
  `docs/SECURITY_MODEL.md` frames redaction as best-effort token hygiene, not a
  guarantee. Not worth a plan.
- **Session state stores raw check stdout/stderr**: investigated and false —
  `src/state/check-state.ts:44-58` stores only redacted, truncated summaries
  (via `truncateForSummary`), never raw output. No finding.
- **`allowDestructiveGitOperations` / `allowForcePush` unread**: folded into plan
  001 as "reserved for future" documentation, not separate findings — the
  features they gate genuinely do not exist yet, so the flags are legitimately
  inert-for-now (the fix is honest docs, covered by 001).
- **No linter/formatter (F4)**: real absence, but a maintainer judgment call
  given `AGENTS.md`'s "no dependencies without a documented rationale" rule. Not
  turned into a plan this round; revisit if the maintainer wants ESLint/Biome.
